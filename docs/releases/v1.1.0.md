# Navne v1.1.0

**Released:** 2026-04-12 · [Technical changelog](../../CHANGELOG.md#110---2026-04-12)

---

## What's new

Version 1.1 adds a settings page. The plugin no longer requires WP-CLI to configure — everything you need to get it running is now in the WordPress admin under Settings → Navne.

---

## Settings page

The page has three sections.

**LLM Provider** covers the API connection. You select a provider (Anthropic is the only option for now), enter an API key, and set the model. The key field never echoes a stored value — once saved, the page shows "Key is configured" without exposing the key. Leaving the field blank on a subsequent save preserves the existing key.

**Operating Mode** shows three options — Safe, Suggest, and YOLO — but only Suggest is active. Safe and YOLO are coming in a future release.

**Post Types** controls which content types trigger the entity pipeline. The list is populated from your registered public post types. At least one must be selected.

---

## Post type guard

The pipeline now checks the allowed post types before dispatching an async job. Previously, every published post triggered extraction regardless of type. Now it only fires for the types you've configured.

---

## What's still not in this release

- Org definition list — the most direct path to better entity disambiguation
- Safe and YOLO operating modes
- Public-facing taxonomy archive pages
- Bulk re-indexing

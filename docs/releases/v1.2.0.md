# Navne v1.2.0

**Released:** 2026-04-12 · [Technical changelog](../../CHANGELOG.md#120---2026-04-12)

---

## What's new

Version 1.2 adds the org definition list — the most direct path to better entity disambiguation — and a deduplication pass that keeps the suggestion queue from filling with entities you've already approved.

---

## Org definition list

The LLM prompt now accepts a list of local figures, acronyms, and jargon you define. Without it, "DOE" could mean the federal Department of Energy or the local school district. "The Governor" means nothing without knowing the state or the person. The definition list hands that local knowledge to the LLM before it processes any post.

The field is on the Settings → Navne page, below Post Types. One definition per line, in `Term: Description` format. Lines starting with `#` are treated as comments and ignored on save — useful for organizing the list without affecting what the LLM sees.

**Example:**

```
# Local acronyms and figures
DOE: The local school district, not the federal Department of Energy
Gov. Smith: Governor Jane Smith, incumbent since 2020
FMS: Fairfield Middle School on Route 9
```

If the field is left blank, the prompt falls back to a neutral message so the LLM isn't confused by an empty placeholder.

---

## Suggestion deduplication

When a post is re-saved, the pipeline now filters out any entity you've already approved. Only new suggestions surface. Previously, re-running the pipeline would re-suggest everything, which made the sidebar noisy and offered no useful signal for posts you'd already reviewed.

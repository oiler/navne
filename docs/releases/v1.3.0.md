# Navne v1.3.0

**Released:** 2026-04-12 · [Technical changelog](../../CHANGELOG.md#130---2026-04-12)

---

## What's new

Version 1.3 activates all three operating modes. The setting on the Settings → Navne page now actually changes how the plugin behaves — something the previous release shipped with the UI in place but the logic deferred.

---

## Safe mode

No pipeline dispatch on save. The sidebar shows a "Process this article" button instead of the spinning loader. Click it to run the pipeline on demand. Results surface the same way as Suggest mode — everything goes to the suggestion queue for manual review before any link appears.

This is the right default for shops where editors own the tagging process entirely and don't want the system doing anything automatically.

---

## Suggest mode

Unchanged from before, now properly labeled. The pipeline dispatches automatically on save, the sidebar polls until results arrive, and editors approve or dismiss suggestions one at a time. Nothing links until a human approves it.

---

## YOLO mode

Entities at 75% confidence or above are approved and linked automatically without any sidebar action. The taxonomy term is created if it doesn't exist yet, and the entity-linking cache for that post is cleared so the new link shows up on the next page load. Entities below 75% still appear as pending suggestions in the sidebar for manual review.

The threshold isn't configurable in this release. 75% reflects the confidence range where the LLM has demonstrated consistent accuracy in practice.

---

## What's still not in this release

- Public-facing taxonomy archive pages
- Bulk re-indexing
- Per-entry entity type hints in the definition list
- Configurable YOLO confidence threshold
